# DEMO — LATENTE (v1)

Duración: 5–7 minutos  
Formato: CLI (terminal)  
Objetivo: Demostrar encuentros humanos sin GPS, sin ranking y sin presión.

------------------------------------------------------------
1. QUÉ ES LATENTE
------------------------------------------------------------

LATENTE es un sistema que NO busca personas.

No hay:
- feeds
- mapas
- swipes
- ranking
- likes
- boosts

LATENTE solo actúa cuando dos personas YA comparten un patrón real de vida.

Incluso entonces:
- no revela identidades
- no muestra rechazos
- no presiona respuestas

LATENTE aparece solo cuando tiene sentido y desaparece después.

------------------------------------------------------------
2. REQUISITOS PARA LA DEMO
------------------------------------------------------------

Antes de ejecutar la demo, el proyecto debe estar correcto:

npm install
npm run test:unit
npm run test:integration

Todos los tests deben pasar.

------------------------------------------------------------
3. ARRANQUE DE LA DEMO
------------------------------------------------------------

Iniciar el CLI:

npm run cli

Mensaje clave para explicar:
“Aquí no hay nada que mirar. Solo decisiones que tomar.”

------------------------------------------------------------
4. FLUJO PRINCIPAL (HAPPY PATH)
------------------------------------------------------------

En el menú, seleccionar:

13) Demo guiada (happy path)

El sistema ejecuta automáticamente:

1. Ingesta de eventos abstractos (sin GPS, sin timestamps exactos)
2. Detección de patrones recurrentes
3. Detección de co-presencia latente
4. Propuesta de ventana futura
5. Aceptación por ambas partes
6. Activación de ventana
7. Confirmación mutua
8. Revelación

Mensaje clave durante la ejecución:

“El sistema no sabe dónde están estas personas.
No sabe quiénes son.
Solo detecta que repiten un mismo patrón de vida.”

------------------------------------------------------------
5. REVELACIÓN (MOMENTO CLAVE)
------------------------------------------------------------

Después de la demo, entrar en:

10) Ver revelaciones

Se muestra:
- una revelación activa
- sin ubicación
- sin mapa
- sin historial
- con fecha de expiración

Mensaje clave:

“La identidad solo aparece cuando ambos confirman.
Si no, el sistema olvida.”

------------------------------------------------------------
6. QUÉ NO HACE LATENTE
------------------------------------------------------------

LATENTE NO:

- muestra rechazos
- penaliza no responder
- compara usuarios
- optimiza engagement
- guarda datos innecesarios

Mensaje clave:

“LATENTE está diseñado para desaparecer.”

------------------------------------------------------------
7. FRASE DE CIERRE
------------------------------------------------------------

LATENTE no quiere tu atención.
Quiere aparecer solo cuando tiene sentido.

------------------------------------------------------------
8. ESTADO DEL PRODUCTO
------------------------------------------------------------

Versión: v1 funcional  
Arquitectura: monolito modular  
Persistencia: in-memory (MVP)  
Privacidad: anti-GPS, anti-tracking, anti-ranking  
Estado: demo-ready

------------------------------------------------------------
9. COMANDOS ÚTILES
------------------------------------------------------------

npm run cli
npm run test:unit
npm run test:integration
